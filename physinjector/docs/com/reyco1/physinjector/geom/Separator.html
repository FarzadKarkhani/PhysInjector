public function Separate(body:b2Body, fixtureDef:b2FixtureDef, verticesVec:Vector.<b2Vec2>, scale:Number = 30):void

Separates a non-convex polygon into convex polygons and adds them as fixtures to the body parameter. There are some rules you should follow (otherwise you might get unexpected results) :

  • This class is specifically for non-convex polygons. If you want to create a convex polygon, you don't need to use this class - Box2D's b2PolygonShape class allows you to create convex shapes with the setAsArray()/setAsVector() method.
  • The vertices must be in clockwise order.
  • No three neighbouring points should lie on the same line segment.
  • There must be no overlapping segments and no "holes".

Parameters

body:b2Body — The b2Body, in which the new fixtures will be stored.
 
fixtureDef:b2FixtureDef — A b2FixtureDef, containing all the properties (friction, density, etc.) which the new fixtures will inherit.
 
verticesVec:Vector.<b2Vec2> — The vertices of the non-convex polygon, in clockwise order.
 
scale:Number (default = 30) — [optional] The scale which you use to draw shapes in Box2D. The bigger the scale, the better the precision. The default value is 30.

public function Validate(verticesVec:Vector.<b2Vec2>):int

Checks whether the vertices in verticesVec can be properly distributed into the new fixtures (more specifically, it makes sure there are no overlapping segments and the vertices are in clockwise order). It is recommended that you use this method for debugging only, because it may cost more CPU usage.

Parameters

verticesVec:Vector.<b2Vec2> — The vertices to be validated.

Returns
int — An integer which can have the following values:
  • 0 if the vertices can be properly processed.
  • 1 If there are overlapping lines.
  • 2 if the points are not in clockwise order.
  • 3 if there are overlapping lines and the points are not in clockwise order.
